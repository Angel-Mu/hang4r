hang4r 1.0 is out — your agents, in parallel

July 17, 2026 · 3 min read

hang4r 1.0 is available for download — a free macOS agents window that runs your Claude Code, Codex, and Cursor CLI agents in parallel. Signed and notarized by Apple's standards, auto-updating, and built on one non-negotiable idea: your subscriptions, your machine. hang4r spawns the CLIs you already have installed and logged in. No API keys, no markup, nothing to pay.

What's in 1.0

  • Parallel sessions in one window — tiled agent sessions, each with streaming chat, an editor, a real terminal, and diff review.
  • Isolated git worktrees — every session on its own branch, checkpointed after every turn. Edit a sent message and the conversation rewinds: a true fork on Claude, a real thread rollback on Codex — and on Cursor the UI honestly says it can only append.
  • Diff review that talks back — comment on the changes; your comments become the agent's next prompt.
  • Agents with eyes — every agent gets a hang4r browser CLI to drive the built-in browser: click, read the DOM, check console errors, screenshot. "I fixed the form" ships with evidence.
  • Worktree setup scripts & dev processes — provision each fresh worktree in the background; opt processes into starting with the agent.

Honest by design

When a backend can't do something, hang4r says so instead of faking it. And the same honesty applies to your Mac: hang4r itself requires no special permissions — if a permission prompt ever appears, it's because your agent or terminal command touched that location, the prompt says so in plain words, and denying is always safe.

Get it

Direct download (Apple Silicon), or: 

brew tap angel-mu/hang4r
brew install --cask hang4r

You'll need the CLIs you want to drive, installed and logged in: claude, codex, and/or cursor-agent. Updates arrive in-app, always as your choice.

hang4r is free and stays free — it wraps subscriptions you already pay for, so there's nothing to sell you. If it saves you an afternoon: GitHub Sponsors · Ko-fi. Bugs and ideas → issues.
